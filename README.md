# 校园二手物品交易系统

这是按数据库原理实验报告完成的校园二手物品交易系统核心版。项目使用 Python 标准库提供后端接口和静态页面服务，使用 SQLite 作为关系数据库，内置建表脚本、视图、触发器和演示数据，不需要额外安装依赖。

## 运行方式

```bash
cd ~/campus-market
python3 backend/app.py
```

启动后访问：

```text
http://127.0.0.1:8000
```

首次启动会自动创建数据库文件：

```text
data/campus_market.db
```

## 已实现功能

- 游客：浏览在售物品、按关键字/分类搜索、按发布时间/价格/热度排序、查看求购信息和公告。
- 注册用户：注册、登录、选择用户类型（学生/教职工/校友）、提交校园身份认证、留言、举报。
- 认证用户：收藏物品、发布物品、编辑物品、上架/下架、发布求购、提交订单、取消订单、确认收货、交易评价。
- 卖家：处理订单，支持确认接单、拒绝接单、取消订单。
- 通知中心：订单、留言、评价、认证审核、举报处理结果会自动生成站内通知。
- 管理员：审核用户认证、维护商品分类、维护交易地点、处理举报、强制下架物品、封禁用户、维护公告、查看信誉异常用户和运营统计。

## 数据库实现点

建表脚本位于：

```text
backend/schema.sql
```

主要关系模式包括：

- `Admin`
- `User`
- `Category`
- `Location`
- `Item`
- `Favorite`
- `Wanted`
- `OrderSheet`
- `Message`
- `Review`
- `Report`
- `Announcement`
- `Notification`

实验报告要求的视图已实现：

- `V_Item_Detail`：物品详情综合视图。
- `V_Order_Summary`：订单管理汇总视图。
- `V_Risky_User`：信誉异常用户预警视图。

关键完整性约束也已落地：

- 买家不能购买自己发布的物品。
- 注册用户类型限制为学生、教职工、校友三类；管理员使用独立 `Admin` 表。
- 生成订单时物品由“在售”自动变为“交易中”。
- 订单取消后物品恢复“在售”。
- 交易成功后物品变为“已售出”。
- 只有交易成功的订单才能评价。
- 评价星级限制在 1 到 5。
- 评价后自动更新被评价人的信用积分。
- 同一物品同一时刻只能存在一个进行中或成功订单。
- 用户不能收藏自己发布的物品。
- 订单、留言、评价、认证和举报处理动作会写入通知表，支持未读/已读管理。

## 项目结构

```text
campus-market/
  backend/
    app.py          # 后端接口、静态页面服务、初始化演示数据
    schema.sql      # SQLite 数据库表、索引、视图、触发器
  frontend/
    index.html      # 页面入口
    styles.css      # 页面样式
    app.js          # 前端交互逻辑
    assets/         # 示例物品图片
  data/             # 运行后生成数据库文件
```
